# FitTracker ロードマップ / 残課題

FitTracker（Expo 製フィットネスアプリ）の今後の開発課題をまとめる。
現状は **Phase 1（ローカル完結）** を実装中。技術構成・詳細は [CLAUDE.md](./CLAUDE.md) を参照。

## ✅ 実装済み（Phase 1）

- オンボーディング
  - 目的別の「目安（参考値）」表示（目標ジャンルは強制せず、すべて任意で選択）
  - 体型シルエット（男女別）＋目的別の推奨 BMI/体脂肪ゾーン
  - 目標体重スライダー＋達成期間からの**期間ベース目標カロリー**算出・ペース警告
  - 目標項目（体重 / 体脂肪率 / 種目重量 BIG3 / 習慣）を**全て任意・並列**で設定
  - 生年月日（年/月/日）入力・年齢自動計算
  - 活動量を「頻度 × 強度」に分離、運動環境は複数選択
- ダッシュボード（今日のカロリー進捗バー・目標進捗バー・7日体重グラフ）
- ワークアウト記録 ＋ 休憩タイマー（経過ストップウォッチ付き）
- 食事管理（カロリー・PFC）
- 体型トラッキング（体重・体脂肪・筋肉量、30日グラフ）
- トレーニングプラン（手動作成 ＋ 環境×頻度×目的での**自動提案**）
- 設定／再設定画面（後から変更・オンボやり直し）
- BIG3 等の目標進捗トラッキング（記録から現在値を集計）

## 🔜 残課題

### Phase 2 — Supabase 導入（クラウド化）
- [ ] ユーザー認証（メール / Google）
- [ ] クラウド同期（複数デバイス対応。AsyncStorage → Supabase へ移行、オフライン対応）
- [ ] フレンド機能・進捗共有（アプリ内 SNS 的機能）

### Phase 3 — 収益化
- [ ] 広告枠（ジム・プロテイン・食事宅配サービス）
- [ ] サブスク（Stripe 連携：広告非表示・AI アドバイス・詳細分析）
- [ ] スマート体重計連携（eufy / Withings 等の API。ネイティブ専用機能は Web では分岐して無効化）

### 機能・UX 改善
- [ ] BIG3 進捗の**種目名マッチング精度向上**（表記揺れの正規化。現状は完全一致のみ）
- [ ] ワークアウトの種目名サジェスト（よく使う種目の入力補助）
- [ ] 生年月日のネイティブ DatePicker 化（現状は年/月/日の手入力）
- [ ] 体型シルエットのさらなるリアル化（イラスト差し替え・体脂肪での見た目変化）
- [ ] ダッシュボードに目標体重・体脂肪への**到達見込み（残り日数）**表示
- [ ] 食事提案のバリエーション拡充・食材データベース
- [ ] プロフィール再設定でオンボーディング全項目を編集可能に（現状は主要項目のみ）

### 技術・品質
- [ ] スマホ実機（iOS / Android）での目視確認・ストア配信準備（EAS Build）
- [ ] ユニットテスト（`src/lib/fitness.ts` などの計算ロジック）
- [ ] エラーハンドリング・空状態（Empty State）の磨き込み
- [ ] アクセシビリティ（コントラスト・タップ領域・スクリーンリーダー）

## 優先度メモ

1. **実機確認 → ストア配信準備**（Phase 1 を「実際に使える」状態にする）
2. **Phase 2（クラウド化）** — 複数デバイス・継続利用の基盤
3. **Phase 3（収益化）**
